# v1.0.3 发布记录

> 日期：2026-05-30
> 主题：Windows 兼容性修复 + 空输入优雅退出

## 背景

1. `.cmd` 文件中使用了 Unix 风格的 `>/dev/null`，在 Windows 上报错导致脚本中断。
2. `.cmd` 和 `.command` 在交互模式下，用户直接按回车（空输入）会陷入循环而非退出。

## 修复

1. **Windows `>/dev/null` → `>nul`**：`.cmd` 中 4 处替换，消除 Windows 兼容错误。
2. **空输入优雅退出**：交互模式检测空输入，显示提示后退出，不再循环。

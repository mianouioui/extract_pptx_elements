# v1.1.3 发布记录

> 日期：2026-05-31
> 主题：行为修正 + Windows 闪退修复 + 名称统一

## 背景

v1.1.2 上线后发现三个问题：

1. **`--media-only` 仍导出文本**：文档说「仅提取图片、视频、音频」，但因 `--with-text` 默认开启，文本被一并导出，与文档矛盾。
2. **不加 `--overwrite` 重复运行静默生成副本**：计数器在每次调用时重置，导致 `001_JPG.jpg` 已存在时生成 `001_JPG_02.jpg`，而非跳过已有文件。
3. **Windows `.cmd` 闪退**：文件使用 LF 换行符，Windows CMD 解析失败直接退出。

同时存在「PPTX 元素提取工具」与「PPTX 内容提取器」两个名称混用的问题。

## 修复

### 1. `--media-only` 排除文本（3 文件 × 2 实现 = 6 处）

`main()` 中传入 `with_text=args.with_text and not args.media_only`，`--media-only` 时强制关闭文本导出。

### 2. 无 `--overwrite` 跳过已有文件（3 文件 × 2 实现 = 6 处）

`unique_output_path` / `Get-UniqueOutputPath` 改为 `Path | None` / 可返回 `$null`：文件已存在且 `overwrite=False` 时返回 `None`/`$null`，调用方检测后跳过。

Python 侧：
```python
if candidate.exists():
    return None
```
PowerShell 侧：
```powershell
if (Test-Path -LiteralPath $candidate) {
    return $null
}
```

### 3. `.cmd` CRLF 换行（`.gitattributes` + 全文转换）

- 新建 `.gitattributes`：`*.cmd -text` 阻止 Git 转换换行符，确保 CRLF 保留在仓库中，GitHub 直接下载也能正常运行。
- `extract_pptx_elements.cmd` 全文 LF → CRLF（1609 行）。

### 4. 名称统一为「PPTX 内容提取器」

三套入口（`.py` docstring + argparse 描述、`.command` 头部 + 横幅 + 内嵌 Python、`.cmd` 头部 + PowerShell 横幅 + 内嵌 Python）全部统一为「PPTX 内容提取器」。

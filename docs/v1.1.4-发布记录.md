# v1.1.4 发布记录

> 日期：2026-05-31
> 主题：输出文件命名规则改为 PPT 名称 + 页码

## 背景

v1.1.3 及之前版本使用 `{页码}_{类型标签}.{扩展名}` 格式命名（如 `001_JPG.jpg`、`002_MP4.mp4`）。类型标签（JPG/MP4/CHART 等）冗余，因为文件已按类型分入不同中文文件夹（`图片/`、`视频/` 等）。

参考 `images-to-pptx` 项目的命名风格，改为 PPT 名称 + 页码的组合，更直观地标识文件来源。

## 改动

### 命名格式

| 旧格式 | 新格式 |
|--------|--------|
| `图片/001_JPG.jpg` | `图片/presentation_001.jpg` |
| `图片/001_JPG_02.jpg` | `图片/presentation_001_02.jpg` |
| `视频/001_MP4.mp4` | `视频/presentation_001.mp4` |
| `文本/001_TXT.txt` | `文本/presentation_001.txt` |

同页同类重复仍用 `_02`、`_03` 后缀。不同类型因分入不同文件夹，即使同页同号也不会冲突。

### 代码变更（3 套入口 + PowerShell 回退）

1. **移除 `tag_for` / `Get-TagFor` 函数**：不再需要生成类型标签。
2. **简化 `output_suffix_for` / `Get-OutputSuffix`**：移除 `tag` 参数，无扩展名时默认 `.bin`。
3. **`unique_output_path` / `Get-UniqueOutputPath`**：参数 `tag` → `pptx_stem`，计数器 key 从 `(dir, slide, tag)` 简化为 `(dir, slide)`。
4. **`Resource` 数据类**：移除 `tag` 字段。

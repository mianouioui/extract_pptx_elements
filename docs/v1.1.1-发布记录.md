# v1.1.1 发布记录

> 日期：2026-05-30
> 主题：输出位置优化 + `.cmd` 自包含化

## 背景

1. v1.1.0 之前，默认输出目录在工具所在的文件夹，对拖拽启动场景不够友好。
2. `.cmd` 启动器依赖外部的 `extract_pptx_elements.py` 文件，发送给用户时容易遗漏。

## 改动

1. **默认输出位置优化**：改为在 PPTX 文件旁创建 `pptx_extracted_elements/`，对拖拽启动更友好。
2. **`.cmd` 自包含化**：`extract_pptx_elements.cmd` 改为内嵌 Python 源码，不再依赖外部 `.py` 文件。
3. **`.command` 刷新**：同步最新内嵌核心逻辑。
4. **移除 `scripts/`**：构建辅助脚本不是终端用户所需，清理出仓库。
5. **逐文件错误隔离**：单个 PPTX 提取失败不会中断其他文件的处理。
6. **README 更新**：补充更规范的操作说明。

## 本次改动的文件

- `extract_pptx_elements.py`：默认输出位置逻辑变更 + 错误隔离
- `extract_pptx_elements.cmd`：内嵌 Python 源码 + 自包含化
- `extract_pptx_elements.command`：同步最新内嵌代码
- `README.md`：操作说明更新
- `scripts/`：移除

## 兼容性说明

- 输出目录从「工具所在文件夹」改为「PPTX 所在文件夹旁」。
- `.cmd` 不再需要 `.py` 文件在旁边，可单独发送。

- 多个 PPTX 中一个损坏：其他正常提取 ✓

# v1.2.0 发布记录

> 日期：2026-05-31
> 主题：新增「元素还原器」，一键把修改后的图片写回 PPTX 且严格保持位置

## 背景

提取工具 `extract_pptx_elements` 可以把 PPTX 里的图片、视频等媒体按页导出，方便批量处理（典型场景：批量去水印、修图）。但处理完之后，如何把这些图片放回原来的 PPTX、并让每个元素的位置丝毫不差，一直缺少一个顺手的工具。

v1.2.0 新增配套程序 `restore_pptx_elements`，专门解决「分离 → 批改 → 一键还原」闭环里的最后一步。

## 核心原理

PPTX 本质是一个 ZIP 包：每张图片是包内的一个「部件」（如 `ppt/media/image1.png`），而图片在幻灯片上的**位置和大小记录在幻灯片 XML**（`<a:off>` / `<a:ext>`）里，**不在图片字节里**。

因此还原的做法是：

1. 复制一份原始 PPTX；
2. 按 `manifest.csv` 的记录，把每个媒体部件的字节替换成修改后的文件；
3. 其它所有内容（尤其是幻灯片 XML）原样拷贝。

由于从不改动任何 XML，**所有元素的位置严格保持不变**——这对图片是硬保证。即使修图后图片的像素尺寸或格式变了，显示位置也不会变（PowerPoint 仍按原版式框缩放）。

## 用法

### 工作流程

1. 提取：`extract_pptx_elements.py "演示.pptx"` → `pptx_extracted_elements/`
2. 在 `图片/` 里批量改图（**保持文件名不变**）
3. 还原：`restore_pptx_elements.py` → `演示_restored.pptx`

### 入口

- Python：`restore_pptx_elements.py`（Python 3.8+）
- macOS：`restore_pptx_elements.command`（放在原始 PPTX 和提取文件夹旁边，双击即可一键还原）
- Windows：`restore_pptx_elements.cmd`（与提取工具一致：内嵌 Python 主实现 + PowerShell 回退，免装 Python）

### 选项

| 选项 | 说明 |
|------|------|
| `--pptx 文件` | 指定原始 PPTX；默认据清单自动查找 |
| `-o 文件名` | 输出 PPTX；默认 `<原名>_restored.pptx` |
| `--images-only` | 只写回图片 |
| `--media-only` | 只写回图片 / 视频 / 音频 |
| `--overwrite` | 覆盖已存在的输出 |
| `--dry-run` | 只预演不写文件 |

## 关键设计

- **自动查找**：从清单的文件名推断原始 PPTX 名称，并在提取文件夹旁边 / 当前目录查找；找不到时提示用 `--pptx` 指定。
- **只改有改动的部件**：与原文件一致的素材自动跳过；报告里区分「已写回 / 未改动 / 缺文件」。
- **共用图片智能选择**：同一张图被多页引用时，提取会生成多份副本；只要改其中任意一份，还原会自动采用「被改过的」那一份。
- **防呆**：指向了错误的原始 PPTX（部件都对不上）时，直接报错、不生成文件，避免产出一个无意义的副本。
- **不动文本**：清单里的 `text`（纯文本导出）不会被写回，避免覆盖幻灯片 XML。

## 验证

端到端测试（构造含定位图片的最小 PPTX → 提取 → 改图 → 还原）确认：

- 还原后仅 `ppt/media/image1.png` 改变；
- `ppt/slides/slide1.xml`（即位置）逐字节不变 → 位置严格保持；
- 即使替换图尺寸不同（4×4 → 7×5）也成立；
- 输出仍是有效、可再次提取的 PPTX 包。

边界场景验证：覆盖保护、`--dry-run` 不写文件、无参自动识别、错误 PPTX 不产出、共用图取「改过的副本」、未改动如实报告。

Windows `.cmd`：内嵌 Python 段已逐字节核对与独立 `restore_pptx_elements.py` 一致，并用「按批处理同样的方式提取临时 .py 再运行」跑通端到端；PowerShell 回退段按已验证的 Python 逻辑移植，已做括号/花括号配平等静态检查，但因本机无 PowerShell 环境，**PowerShell 回退路径尚需在真实 Windows 上实测**。还原器从不改动原始 PPTX，故即便回退路径有问题，最坏也只是输出文件不可用，不会损坏原文件。

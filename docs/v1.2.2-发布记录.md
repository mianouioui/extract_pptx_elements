# v1.2.2 发布记录

> 日期：2026-05-31
> 主题：修复提取清单、母版资源、文本导出与共享素材还原冲突处理

## 背景

本次检查发现几个影响真实工作流的边界问题：重复提取可能清空 `manifest.csv`，母版/版式里的可见图片不会被导出，文本 run 会被拆成多行，以及同一共享素材被改成多个不同版本时还原器仍会选择第一份继续写回。

v1.2.2 对这些逻辑做了集中修复，重点是让「提取 → 修改素材 → 还原」链路更稳。

## 修复

### 1. 重复提取不再清空 manifest

不带 `--overwrite` 重复提取时，已有素材仍会被保护，不会覆盖用户已经修改过的文件；但 `manifest.csv` 会按当前 PPTX 的应有资源映射重新生成，不再变成空清单。

这能避免用户在修图后误跑一次提取，导致还原器失去素材到 PPTX 内部部件的对应关系。

### 2. 提取 slideLayout / slideMaster 资源

提取器现在会继续追踪幻灯片关系中的 `slideLayout` 与 `slideMaster`，因此版式或母版里引用的图片也会进入导出结果。

典型场景包括：

- 母版 Logo
- 版式背景图
- 多页共用的装饰图片

### 3. 文本按段落导出

文本导出从「每个 `<a:t>` 单独一行」改为「按 `<a:p>` 段落合并 run」。

PowerPoint 经常会因为局部加粗、颜色或字体变化，把一句话拆成多个 text run。新逻辑会把同一段里的 run 合并，减少不必要的换行和空格丢失。

### 4. 共享素材冲突时停止还原

如果同一 PPTX 内部部件被多页共用，提取时会出现多份副本。以前当多份副本都被修改、且内容不一致时，还原器会警告但采用第一份继续生成文件。

现在遇到这种冲突会直接停止还原并提示冲突文件，避免生成一个用户未明确选择的结果。

## 验证

已通过以下验证：

- Python 语法检查：`py_compile`
- 临时 PPTX 边界测试：
  - 重复提取不会清空 `manifest.csv`
  - 已有素材不被覆盖
  - `slideLayout` / `slideMaster` 图片可被提取
  - 文本 run 会按段落合并
  - 共享素材多份副本改成不同内容时还原失败且不写输出
  - 共享素材只改一份时仍可正常还原
- macOS `.command --version` 冒烟测试通过
- 四个启动器内嵌 Python 与独立 `.py` 一致
- Windows `.cmd` 仍保持 CRLF 行尾
